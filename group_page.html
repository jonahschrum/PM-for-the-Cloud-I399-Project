Query Automation Cloud System:

A New Way to Automate Query Operations



Project Objective

The objective of this project is to design and partially implement the tools necessary to create an online website that can take user's inputted code and queries and can return data to the user from an external third party (data warehouse, etc.) that provides continual and updated results to the user. This allows the user to maintain present and up-to-date data without the user having to manually update or check the queries. The website can also format data, whether in be statistical graphs, pie charts, or other traditional layouts. The project will be designed and will have workable cloud based components by mid December, however whether the project will ever become fully functional depends on outcomes of design feasibility.







Our system would primarily run using AWS containers, as well as use relational databases for the majority of the data stored. This is because our software relies on security and data accuracy, being that queries are often very confidential and/or critical to the infrastrcuture of the person using it. In this case, reltational databases serve best in processing accurate and secure data at the expenes of less speed of data retrieval.





Database format

Data will be hosted on relational data bases and on the website that we created for this service. The website will have acceess to the relational databases as well as other information. For example, one link may take you to the query that was just uploaded and another link will be for a FAQ page. In addition, our webpage will be a one stop shop so to say. So the customer would be able to acomplish their professional tasks on one page but will have plenty of step by step images, videos, and audio files on how to best use our program. This will also make it easier for the client and they will be able to see the full capability of our website. There will be direct links and pages dedicated to tutorial and additional information. In addition, there will be a lot of customization options for the user to customize their data and there will be a automated recomendations from the program suggesting how the client should breakdown their specific dataset. This is best seen in the microserviecs section.







Microservices

Color Customization Upgrade
The customer has the option to pay for additional colors for their data to further customize its appearence. Only a few colors are avalible in the free package and having more colors allows the user to break their data down further and deliver more insight.
Graph Customization Upgrade
The customer has the option to pay for additional graph and chart styles. This add on package includes more variety of charts and graphs that will be able to load and organize more information than the free teir charts and graphs. In addition, the user would be able to access more tutorials relating to using the new graphs offered in this package.
Automated Recomendations
For an additional fee the customer has the option to add the automated recomendations package. This package is a computer generated solution on what charts your data needs and how the information should be organized. This package does not customize colors, but rather the data. This can cut down devloper time and accelerate the rate of projects.
